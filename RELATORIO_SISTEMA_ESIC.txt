1. APRESENTACAO
================================================================================

O presente relatorio descreve o Sistema e-SIC (Servico de Informacao ao
Cidadao) implantado para a Prefeitura Municipal de Rio Claro - RJ,
contemplando sua finalidade legal, arquitetura tecnologica, funcionalidades

para a gestao municipal.

================================================================================
2. BASE LEGAL E FINALIDADE
================================================================================

O sistema foi desenvolvido em atendimento a Lei Federal no 12.527/2011 -
Lei de Acesso a Informacao (LAI), que obriga todos os orgaos e entidades da
Administracao Publica a garantir o direito de acesso a informacao a qualquer
cidadao.

A LAI estabelece:

  - Prazo maximo de 20 dias corridos para resposta ao pedido, prorrogavel
    por mais 10 dias mediante justificativa.
  - Possibilidade de 3 instancias de recurso em caso de negativa ou resposta
    insatisfatoria.
  - Obrigatoriedade de portal de transparencia com informacoes publicas
    accessiveis sem cadastro.

O e-SIC digitaliza integralmente esse fluxo, eliminando o atendimento
presencial e garantindo rastreabilidade e conformidade legal em cada etapa.

3. VISAO GERAL DO SISTEMA
================================================================================

O sistema e uma plataforma web de uso exclusivamente municipal, configurada
com a identidade visual e os dados da Prefeitura de Rio Claro - RJ (brasao,
favicon, cor institucional, CNPJ, endereco e contatos). Ele opera 24 horas
por dia, 7 dias por semana, e e acessivel por qualquer dispositivo com
navegador de internet.

-----------------------
  Sistema (producao)     : A configurar conforme hospedagem
  Portal de Transparencia: Pagina inicial - acesso publico sem login

4. ARQUITETURA TECNOLOGICA
================================================================================

O sistema e composto por duas camadas independentes que se comunicam
via API REST.

4.1 Backend (Servidor de Aplicacao)
------------------------------------
  Linguagem        : TypeScript / Node.js
  Framework        : Express 5
  Banco de dados   : PostgreSQL 14+
  ORM              : Prisma 5.21
  Autenticacao     : JSON Web Token (JWT)
  Upload de arquivos: Multer (armazenamento local)
  E-mail           : Nodemailer (SMTP configuravel)
  Porta padrao     : 3001

4.2 Frontend (Interface do Usuario)
-------------------------------------
  Linguagem        : TypeScript / React 19
  Build            : Vite
  Roteamento       : React Router v7
  Icones           : Lucide React
  Porta padrao     : 5173 (dev) / 80 (producao)

4.3 Diagrama Simplificado
--------------------------

  Cidadao/Servidor --> Frontend (React) --> Backend (Node.js) --> PostgreSQL
                                                  |
                                                  +-- /uploads (brasao, favicon, anexos)
                                                  +-- SMTP     (notificacoes por e-mail)

5. MODULOS E FUNCIONALIDADES
================================================================================

5.1 Portal Publico (sem login)
-------------------------------
  Pagina inicial         : Apresentacao institucional com slogan e identidade
  Portal de Transparencia: Busca textual em pedidos respondidos
  Cadastro de cidadao    : Registro com CPF/CNPJ, e-mail, telefone e senha
  Login                  : Autenticacao por e-mail e senha com token JWT
  Recuperacao de senha   : Tela de recuperacao disponivel

5.2 Painel do Cidadao
----------------------
  Novo pedido de acesso  : Formulario com descricao e nivel de sigilo
                           (Reservado, Secreto, Ultrassecreto)
  Acompanhamento         : Listagem de todos os pedidos com status em tempo real
  Detalhe do pedido      : Historico de tramitacao, datas e resposta completa
  Upload de anexos       : Envio de documentos de suporte ao pedido
  Recurso                : Abertura em 1a, 2a e 3a instancia

5.3 Painel do Tecnico / Servidor
----------------------------------
  Lista de pedidos       : Pedidos encaminhados ao setor do servidor
  Responder pedido       : Registro de resposta com prazo controlado
  Triagem de pedidos     : Classificacao e encaminhamento ao departamento
  Historico de tramitacao: Registro completo de cada etapa, usuario e data

5.4 Painel Administrativo
--------------------------
  Dashboard              : Total de pedidos, abertos, respondidos, vencidos,
                           recursos
  Gestao de pedidos      : Visualizacao, triagem e encaminhamento geral
  Encaminhamento WhatsApp: Botao gera mensagem pre-formatada para WhatsApp
                           do servidor responsavel
  Gestao de usuarios     : Criar, editar e excluir servidores com perfis
  Gestao de departamentos: Criar, editar e excluir secretarias/setores
  Configuracoes          : Brasao, favicon, cor, CNPJ, slogan, endereco

6. PERFIS DE ACESSO
================================================================================

O sistema possui 5 perfis distintos, com permissoes graduadas:

  CITIZEN (Cidadao)
    Quem usa : Municipipes em geral
    Acesso   : Abrir pedidos, acompanhar, enviar recursos

  TECHNICIAN (Tecnico)
    Quem usa : Servidores das secretarias
    Acesso   : Responder pedidos atribuidos ao seu setor

  AUTHORITY (Autoridade)
    Quem usa : Secretarios / Chefes de setor
    Acesso   : Responder e acompanhar pedidos do setor

  CONTROL (Controle)
    Quem usa : Controladoria / Ouvidoria
    Acesso   : Visualizar todos os pedidos e usuarios; monitorar prazos

  ADMIN (Administrador)
    Quem usa : TI / Gestor do e-SIC
    Acesso   : Total - configurar, gerenciar, encaminhar, deletar

7. FLUXO DE UM PEDIDO DE ACESSO A INFORMACAO
================================================================================

  1. Cidadao cadastra-se (se necessario) e faz login

  2. Cidadao preenche e envia o pedido
       --> Sistema gera protocolo unico (ex: ESIC-2026-001)
       --> Registra prazo de 20 dias corridos

  3. Administrador ou Controle recebe o pedido com status ABERTO

  4. Pedido e encaminhado ao departamento competente
       --> Sistema registra movimento
       --> Notifica via WhatsApp e/ou e-mail

  5. Tecnico ou Autoridade do setor responde o pedido
       --> Status muda para RESPONDIDO

  6. Cidadao visualiza a resposta no painel ou por e-mail

  7. Se insatisfeito -> Cidadao abre recurso (1a instancia)
       --> Controle/Autoridade analisa e registra decisao

  8. Se ainda insatisfeito -> 2a e 3a instancias disponiveis

================================================================================
8. TRANSPARENCIA PUBLICA
================================================================================

A pagina de Transparencia e acessivel sem qualquer login. A plataforma
permite que qualquer cidadao:

  - Busque pedidos respondidos por palavra-chave
  - Veja o numero do protocolo, secretaria responsavel, data, pergunta
    e resposta
  - Clique em temas sugeridos para acelerar a busca

Isso cumpre o paragrafo 3o do Art. 8o da LAI, que exige divulgacao proativa
das informacoes de interesse publico.

================================================================================
9. SEGURANCA E CONFORMIDADE
================================================================================

  Autenticacao         : JWT com expiracao configuravel; senha nunca trafega
                         em texto claro
  Senhas               : Hash irreversivel com bcrypt (salt 10); validacao de
                         forca no cadastro (8+ chars, maiuscula, minuscula,
                         numero, simbolo)
  Autorizacao          : Middleware de perfil em todas as rotas sensiveis
  Upload de arquivos   : Filtro por tipo MIME; limite de 2 MB por arquivo
  Dados sensiveis      : CPF/CNPJ vinculados ao usuario; nao expostos
                         publicamente
  Controle de acesso   : RBAC (Role-Based Access Control) em todas as rotas
  Prevencao de injecao : Prisma ORM com queries parametrizadas (sem SQL bruto)

16. CONCLUSAO
================================================================================

O sistema e-SIC entregue a Prefeitura Municipal de Rio Claro - RJ e uma

A plataforma coloca Rio Claro - RJ em conformidade integral com a LAI,
moderniza o atendimento ao cidadao, elimina o fluxo presencial e manual de
pedidos de informacao, e oferece rastreabilidade total de cada solicitacao,
do protocolo a resposta final.
